## Warum lizenzfreie Bilder verwenden?

1. **Urheberrecht** — geschützte Bilder ohne Genehmigung = Rechtsverletzung
2. **Rechtliche Konsequenzen** — Abmahnungen, Schadenersatz vermeiden
3. **Authentizität** — einzigartige Bilder, nicht dieselben wie auf 1.000 anderen Sites
4. **Konflikte vermeiden** — Nutzungsbedingungen genau prüfen
5. **Quellenangaben** — manche Lizenzen erfordern Urhebernennung

---

## Lizenzfrei vs. geschützt

| | Lizenzfrei | Geschützt |
|---|---|---|
| Nutzung | Frei nutzbar (mit Lizenzbedingungen) | Nur mit ausdrücklicher Genehmigung |
| Kosten | Kostenlos (oft) | Lizenz kaufen |
| Quellenangabe | Je nach Lizenz nötig | Je nach Vereinbarung |
| Typisch | Unsplash, Pexels, Pixabay | Shutterstock, Getty |

**Faustregel:** Immer Nutzungsbedingungen lesen. Bei CC0 = komplett frei. Bei CC BY = Urheber nennen.

## Tipps für die Bildauswahl

- **Passend zum Thema** — unterstützt die Botschaft, keine generischen Stock-Fotos
- **Hohe Qualität** — hochauflösend, klar auf allen Bildschirmgrößen
- **Konsistenz** — gleicher Stil, Farbpalette, visuelle Sprache auf der ganzen Site
- **Relevanz** — Bild = ergänzt den Text, nicht Dekoration
- **Originalität** — eigene Fotos wenn möglich, hebt von anderen ab
- **Emotionale Wirkung** — positive Emotionen wecken, Verbindung zur Zielgruppe
- **Ladezeit** — Bilder komprimieren (WebP, max. 200KB für Hero)
- **Lizenz beachten** — CC0 = frei, CC BY = Urheber nennen, immer prüfen

---

## Wo Bilder auf einer Landing Page platzieren

1. **Hero / Header** — erster Eindruck, zeigt Produkt in Aktion
2. **Produktfotos** — echtes Produkt, keine Illustrationen (Onepage-Regel: realistisch)
3. **Blog-Beiträge** — Text ergänzen, Lesbarkeit verbessern
4. **Galerien / Portfolios** — eigene Arbeit präsentieren
5. **Social Media Posts** — Reichweite und Engagement steigern

## Anwendung auf pondex_

**Was pondex_ aktuell nutzt:**
- Hero Score Card: inline JSX (React) — keine externen Bilder nötig ✅
- LogosBar: nur Textlogos — korrekt (Yahoo Finance, SEC EDGAR, Groq AI) ✅
- Testimonials: Initial-Avatare (Buchstaben) — kein Foto nötig, authentischer ✅

**Empfehlung für Gründerhinweis (FounderNote):**
- Echtes Foto von Daniel → erhöht Authentizität deutlich
- Quelle: eigenes Foto oder professionelles LinkedIn-Bild
- Format: quadratisch, 400×400px, WebP, <100KB
- Bis echtes Foto verfügbar: Initial-Avatar "D" in C[600] (wie aktuell implementiert)

**Empfehlung für zukünftige Sections:**
- Unsplash für emotionale Hintergrundbilder (z.B. Investor am Laptop)
- Keine generischen Stock-Fotos (Mann im Anzug mit Diagramm)
- Wenn möglich: Screenshots/Recordings der echten App statt Mockups
